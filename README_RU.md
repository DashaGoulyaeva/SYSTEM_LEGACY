# SYSTEM_LEGACY

> Браузерный инкрементал про наблюдение, износ и память, которая остаётся внутри старых систем.

`SYSTEM_LEGACY` — это сольная narrative incremental-игра на vanilla `JavaScript`, `HTML` и `CSS`.
Игрок приходит на уже идущую дежурную смену, следит за системным журналом, реагирует на сбои, стабилизирует разваливающиеся процессы и постепенно спускается всё глубже по слоям старой архивной машины.

Проект специально строится на сухом, техническом первом впечатлении.
История не выкладывается сразу. Она проступает через повторение действий, поведение системы, названия, темп и напряжение между рутинным обслуживанием и чем-то более личным, скрытым под этой оболочкой.

## Что уже можно потрогать

Текущий прототип сосредоточен на первом играбельном куске:

- вход через журнал, а не через пустой статичный экран
- полноценный поток дежурной смены: инструктаж, сбои, поиск ошибки, разбор, очистка и глубокая дефрагментация
- модель слоёв с проработанными первыми тремя слоями и общей структурой на девять
- раздельные ресурсы `стабильность`, `память`, `наблюдение` и `знание`
- сохранение и восстановление смены через `localStorage`
- панель поздних улучшений для постоянного прогресса

## Направление проекта

У проекта есть несколько жёстких принципов:

- журнал — главное поле игры
- ранние слои должны оставаться сухими, техническими и слегка негостеприимными
- смысл должен рождаться из наблюдения, а не из прямого объяснения
- в основе это всё ещё инкрементал, даже когда нарратив станет плотнее

## Технологии

- Vanilla `JavaScript`
- `HTML`
- `CSS`
- без фреймворков
- без отдельного build step

## Как запустить локально

Открой [index.html](./index.html) в браузере.

Сейчас проект рассчитан прежде всего на локальную игру, пока ранние слои собираются, балансируются и доводятся до рабочего состояния.

## Текущий статус

`SYSTEM_LEGACY` находится в активной разработке.
Сейчас главная цель — довести первый слой до состояния настоящей игровой петли:

- игрок входит в уже живую систему, а не в пустой интерфейс
- журнал первым зовёт к действию, а кнопки отвечают на него
- базовый инкрементальный ритм читается и держится
- первые слои работают сами по себе ещё до углубления в поздний нарратив
