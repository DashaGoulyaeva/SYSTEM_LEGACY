# Resource Strategy

## Simple Work

- One role.
- One checklist.
- One validation pass.

## Medium Work

- One lead role.
- One builder role.
- One checker role.

## Complex Work

- Split into architecture, implementation, QA, and review.
- Add docs if the result is user-visible or reusable.
- Run parallel work only on disjoint file ownership.

## High-Risk Work

- Keep each step small.
- Validate after each slice.
- Prefer the safest path over the shortest path.

## Archive-Only Work

- One archivist role.
- One vault checklist.
- One validation pass.
- Keep the archive separate from live development notes.
