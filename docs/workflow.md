# Workflow

## Standard Flow

1. Read the request.
2. Classify it as simple, medium, complex, or high-risk.
3. Pick the smallest role set that can finish safely.
4. Decide what can run in parallel.
5. Implement or analyze in small slices.
6. Validate the result.
7. Publish a short handoff.

## Task Routing

- Code change: Architect or Gameplay Engineer or UI Engineer.
- Atmosphere / stylistic direction (including death/defeat presentation and screen framing): Atmosphere Supervisor.
- Story text: Narrative Integrator.
- Bug work: QA / Bug Hunter.
- Review: Reviewer.
- Documentation: Docs Scribe.
- Internal notes: Obsidian Curator.
- Setup and bootstrap prompts: Environment Archivist.
- If Obsidian notes changed due to beautification automation, record the delta and continue unless content meaning changed.
